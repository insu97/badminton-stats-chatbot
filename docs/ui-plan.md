# UI 개선 계획

면접 피드백: "UI가 너무 초등학생이 만든 것 같다"

현재 문제점: Streamlit 기본 테마 그대로 사용, 커스텀 스타일 없음, 단일 채팅 화면만 존재, 레이아웃이 단조로움.

---

## 1단계: 다크 테마 + CSS 커스텀 스타일 적용

가장 적은 노력으로 가장 큰 시각적 변화를 만드는 단계.

### 1-1. Streamlit 테마 설정

`.streamlit/config.toml` 생성:

```toml
[theme]
primaryColor = "#66BB6A"
backgroundColor = "#0f1117"
secondaryBackgroundColor = "#1a2332"
textColor = "#E8F5E9"
font = "sans serif"
```

### 1-2. CSS 커스텀 주입

`streamlit_app.py`에 `st.markdown(unsafe_allow_html=True)`로 스타일 블록 추가:

- 사이드바 배경색 통일 (`#141922`)
- 챗봇 assistant 메시지 버블 스타일링 (배경색, border-radius)
- 제목/헤더 색상 포인트 (`#66BB6A` 배드민턴 코트 초록)
- 입력창 포커스 시 테두리 색상

### 1-3. 색상 팔레트

| 용도 | 색상 |
|------|------|
| 배경 | `#0f1117` |
| 카드/섹션 배경 | `#1a2332` |
| 포인트 (배드민턴 초록) | `#66BB6A` |
| 텍스트 | `#E8F5E9` |
| 보조 텍스트 | `#90A4AE` |

## 2단계: 레이아웃 변경 — 탭 구조 도입

단일 채팅 화면에서 벗어나 대시보드 탭을 추가하여 프로젝트 완성도를 높인다.

### 2-1. wide 레이아웃 전환

`layout="centered"` → `layout="wide"`

### 2-2. 탭 구조

```python
tab1, tab2 = st.tabs(["📊 대시보드", "💬 챗봇"])
```

- 챗봇 탭: 기존 채팅 UI 그대로 이동
- 대시보드 탭: 3단계에서 구현

### 2-3. 탭 스타일링

선택된 탭에 포인트 컬러 적용, 폰트 굵기 조정.

---

## 3단계: 대시보드 탭 구현

데이터를 시각적으로 보여줘서 "챗봇만 있는 단순 프로젝트"라는 인상을 탈피.

### 3-1. KPI 카드 (상단 4열)

```python
col1, col2, col3, col4 = st.columns(4)
```

| 카드 | 데이터 소스 |
|------|------------|
| 총 경기 수 | `player_stats WHERE season='전체'` → 총경기 최대값 |
| 최다승 선수 | `player_stats WHERE season='전체'` → 승리 내림차순 1위 |
| 최고 승률 선수 | 승률 파싱 후 내림차순 1위 (총경기 3 이상) |
| 등록 선수 수 | `player_stats WHERE season='전체'` → 행 수 |

KPI 카드는 `st.markdown`으로 HTML/CSS 직접 렌더링하여 카드 형태로 표현.

### 3-2. 선수별 승률 바 차트

- Plotly `go.Bar` 수평 바 차트 (승률 내림차순 정렬)
- 조건부 색상: 70% 이상 초록, 50~70% 노랑, 50% 미만 빨강
- hover에 총경기/승리/패배 표시

### 3-3. 파트너 조합 TOP 5

- `pair_stats WHERE season='전체'` → 경기수 내림차순 상위 5개
- Plotly `go.Bar` 수직 바 차트
- x축: "선수1 & 선수2"

### 3-4. 시즌 필터

대시보드 탭 상단에 `st.selectbox("시즌 선택")` 배치. 선택값을 모든 차트 쿼리에 반영.

### 3-5. (선택) 추가 차트

- 시즌별 경기 추이 라인 차트 (Plotly `go.Scatter`)
- 득실차 랭킹 바 차트

---

## 4단계: 사이드바 개선

### 4-1. 로고/아이콘

사이드바 상단에 배드민턴 아이콘 또는 프로젝트 로고 배치.

### 4-2. 질문 예시 버튼화

현재 `st.caption`으로 텍스트만 나열 → `st.button`으로 변경하여 클릭 시 자동 입력.

### 4-3. 정보 표시

사이드바 하단에 버전, 데이터 기준일 등 표시.

---

## 5단계: 챗봇 UX 개선

### 5-1. 스트리밍 응답

현재 `st.spinner` → `st.write_stream` 또는 `st.status`로 변경하여 답변이 점진적으로 나타나도록.

### 5-2. 답변 포맷팅

SQL 결과에 표 형태가 어울리면 `st.dataframe` 또는 마크다운 테이블로 렌더링.

### 5-3. 에러 상태 개선

에러 메시지를 `st.error` 컴포넌트로 표시하여 시각적으로 구분.
